# Repository Guidance

## Compatibility

- This plugin supports both Vim and Neovim. New user-facing features must be
  evaluated and tested on both editors unless they fundamentally require a
  platform-specific API.
- Neovim Lua features target Neovim 0.11 or later.
- Vim's interactive resize mode and built-in window chooser require
  `+popupwin`. Report a clear error when that feature is unavailable.
- Preserve the existing Vimscript configuration variables. Neovim Lua
  configuration may use them as defaults.

## README media

- Do not remove existing screenshots or GIFs unless the documented feature no
  longer exists, its interaction has been completely replaced, or the user
  explicitly requests removal.
- The following media must remain while their corresponding features exist:
  - `resize_mode.gif`
  - `toggle_zoom.gif`
  - `vsplit_new.gif`
  - `vsplit_cur.gif`
  - `split_new.gif`
  - `split_cur.gif`
  - `hist_msg.gif`
  - `break_cur.gif`
- `choose_win.gif` documents the old external `vim-choosewin` interaction. It
  does not need to be restored after the built-in chooser replaces that flow.

## Window chooser

- Vim and Neovim both provide a built-in chooser.
- Keep the default pane labels and behavior aligned across both implementations.
- `large_glyphs` and the matching Vimscript table contain static A-Z output
  generated by `toilet` for the `smblock` and `pagga` fonts. Pane labels come
  from the preset character sequence, then retrieve their glyph from the
  configured font table. Preserve generated glyphs unless the user explicitly
  requests regeneration. Size each marker to its glyph dimensions and keep
  the rounded border and `IncSearch`-style highlighting. Pad every rendered
  glyph line so the character is horizontally centered in a consistent marker
  width for the selected font.
- `c`, `d`, `g`, `q`, and digits are reserved and excluded from pane labels.
  Digits `1` through `9` select tabs when multiple tabs are open.
- Explicit user configuration containing `C`, `D`, `G`, or `Q` must raise an
  error instead of silently filtering the value.
- Pane labels must accept both lowercase and uppercase input.
- See `KNOWN_ISSUES.md` before changing chooser input handling.

## Documentation and generated files

- Keep `README.md` and `doc/vim-tmuxlike.txt` aligned with current behavior.
- Regenerate `doc/tags` with `:helptags doc` after changing help tags.
- Do not commit runtime artifacts such as `nvim.log`, `.swp`, `.swo`, or
  editor backup files.
- Preserve unrelated user changes in the working tree.

## Verification

Run these checks after relevant changes:

```sh
nvim --headless -u NONE -i NONE -S test/neovim.vim
HOME=/tmp/vim-tmuxlike-home vim -Nu NONE -n -es -S test/vim.vim
for f in lua/tmuxlike/*.lua; do luac -p "$f" || exit; done
git diff --check
```

After help changes, also run:

```sh
nvim --headless -u NONE -i NONE -c 'helptags doc' -c 'qa!'
```
